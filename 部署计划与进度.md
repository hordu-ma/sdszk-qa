# 部署计划与进度

## 1. 目标与现状

- 目标：将 `clinic-sim` 部署到 B 服务器，模型使用 `qwen3-30b-a3b`，并完成 API 可用性验证。
- 当前限制：B 服务器暂未开通外网镜像下载权限，`docker pull` / `compose up` 拉取 `vllm/vllm-openai:latest` 超时失败。
- 当前策略：采用“双路径部署”——有网络时走 Docker 独占模型；无网络时走 systemd 模型 + Docker API。

---

## 2. 完整部署方案

### 2.1 方案 A（推荐）：Docker 独占 vLLM（B 服务器可拉取镜像时）

1. 本地准备

- 代码与编排：使用仓库当前版本（已包含 `prod-b.yml` 调整）。
- 生产环境变量：使用 `prod.env`（上传后命名为 B 服务器 `.env`）。

2. B 服务器部署

- 禁用旧 systemd 模型服务（防冲突）：
  - `sudo systemctl disable --now qwen3-30b-a3b.service`
- 启动服务：
  - `docker compose --env-file /opt/clinic-sim/.env -f src/infra/compose/prod-b.yml up -d --build postgres minio vllm api`
- 迁移数据库：
  - `docker compose --env-file /opt/clinic-sim/.env -f src/infra/compose/prod-b.yml exec api uv run alembic -c src/apps/api/alembic.ini upgrade head`

3. 验收

- `curl http://127.0.0.1:8000/health` 返回 200
- `curl http://127.0.0.1:8001/v1/models` 包含 `qwen3-30b-a3b`
- `docker compose ... ps` 全部健康运行
- `docker compose ... logs --tail=200 api|vllm` 无持续报错

### 2.2 方案 B（受限网络下可落地）：systemd 模型 + Docker API

1. 启动模型服务（systemd）

- `sudo systemctl enable --now qwen3-30b-a3b.service`
- `curl http://127.0.0.1:8001/v1/models`

2. 启动数据库与 API（不启动 vllm 容器）

- `docker compose --env-file /opt/clinic-sim/.env -f src/infra/compose/prod-b.yml up -d --build --pull never postgres minio api`

3. 环境变量要求

- `.env` 中必须配置：
  - `LLM_BASE_URL=http://<B_SERVER_IP>:8001`（或 API 可达的本机地址）
  - `LLM_MODEL=qwen3-30b-a3b`

4. 验收

- `curl http://127.0.0.1:8000/health` 返回 200
- 对话接口可正常调用模型

### 2.3 方案 C（离线镜像导入，供后续使用）

在可联网机器执行：

- `docker pull vllm/vllm-openai:latest`
- `docker pull postgres:15-alpine`
- `docker pull minio/minio:latest`
- `docker save -o clinic-sim-images.tar vllm/vllm-openai:latest postgres:15-alpine minio/minio:latest`

传到 B 服务器后执行：

- `docker load -i clinic-sim-images.tar`
- 再按“方案 A”启动。

## 5. 回滚预案（保底）

- 配置回滚：恢复 `.env.bak.<timestamp>`。
- 模型方式回滚：
  - Docker → systemd：停 `vllm` 容器后 `systemctl enable --now qwen3-30b-a3b.service`
  - systemd → Docker：`systemctl disable --now qwen3-30b-a3b.service` 后起 `vllm` 容器
- 服务回滚：回退至上一个发布包后 `docker compose up -d`。
