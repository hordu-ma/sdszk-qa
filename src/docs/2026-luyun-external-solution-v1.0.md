# 鲁韵思政大模型建设方案

<p align="center"><strong>面向山东省大中小学思政课一体化建设的教学智能支持平台</strong></p>

<p align="center"><strong>版本：v1.0 对外方案</strong></p>
<p align="center"><strong>日期：2026 年 7 月</strong></p>
<p align="center"><strong>适用对象：建设单位、使用单位、运营单位、信息化管理部门及项目专家组</strong></p>

> 本方案用于项目汇报、合同附件和建设沟通。阶段范围、工程顺序和验收细则以《鲁韵思政大模型：教学与课程论驱动的分阶段开发计划》v1.0 签字版为唯一依据。

## 一、方案摘要

鲁韵思政大模型面向山东省大中小学思政课教师、教研员、专家和管理人员，建设“可信资料支撑、教学成果生产、证据化诊断、一体化教研、资源运营和安全治理”一体化的教学智能支持平台。

项目不把“部署一个开源大模型”视为建设完成，也不采用“一段提示词直接生成整份教案”的简单方式。平台以正式课程文件和授权资料为依据，以教学与课程论结构、专家规则和受控任务流程为约束，以经专项评测的本地模型为生成能力，并通过教师修改、专家审核、版本追溯和持续评测形成质量闭环。

项目最终形成两个核心价值：

- 教师能够更快查到可信依据，形成可编辑、可诊断、可局部修改、可导出的教学成果。
- 中心能够管理专业标准、资料、资源审核、质量、安全和运行，而不只是管理模型和问答次数。

## 二、建设背景

山东省大中小学思政课一体化建设需要同时解决课程依据权威、学段递进清晰、课堂设计可实施、教研成果可沉淀和平台运行可治理等问题。现有问答基础版已经具备登录、主题问答、流式输出、历史会话和基础审计能力，但仍缺少专业教学成果对象、可信资料引用、证据化诊断、版本修改、资源审核和生产运营能力。

本次建设将平台从“问答工具”升级为“思政教学智能支持工作台”，优先解决教师真实工作中的四类高频任务：

1. 查政策、课标、教材和中心资料依据。
2. 形成一节课或一个单元的教学设计。
3. 诊断并修改已有教案。
4. 生成、保存、复用和导出课堂及教研成果。

## 三、建设目标与原则

### 3.1 建设目标

- 建设权属清晰、版本有效、可定位引用的思政课程知识体系。
- 建立课程层、单元层、课时层和实施反思层的教学成果结构。
- 建设教学设计生成、诊断、局部修改、版本对比和标准导出闭环。
- 支持小学、初中、高中、大学四学段及大中小学一体化专题设计。
- 建设网页端完整教学工作台和小程序轻量端。
- 建立资料、模型、规则、任务、成果、审核和运行的全链路追溯。
- 支持校内私有化部署，形成可监控、可降级、可恢复、可回滚的运行体系。

### 3.2 建设原则

- **育人为本：** AI 提供依据、草稿和建议，教师和专家保留最终教学判断。
- **专业约束：** 课程标准、教材、专家量规和教学结构共同约束模型输出。
- **可信可追溯：** 政策和教材相关结论展示来源，资料不足时明确提示。
- **成果导向：** 主对象是可编辑、可版本化的教学成果，不是无边界聊天记录。
- **安全可控：** 权限最小化、生成内容有标识、公开发布有人审、重大风险可停用。
- **分阶段建设：** 先完成一个纵向样板，再扩展四学段、运营和增强能力。

## 四、服务对象与身份体系

### 4.1 主要用户

| 用户 | 主要任务 |
| --- | --- |
| 一线教师 | 查依据、备课、改教案、生成课堂材料、保存和导出成果 |
| 中心专家/审核员 | 维护专业量规、审核资料、抽检质量、仲裁争议 |
| 教研员/骨干教师 | 单元设计、跨学段专题、同课异构、教研成果沉淀 |
| 中心管理员 | 组织、资源、审核、使用、质量和运行管理 |
| 技术运维人员 | 模型、任务、容量、故障、安全和版本管理 |

合同建设周期内不直接向学生开放生成式问答，不根据教案或使用记录推断教师、学生的政治态度、能力等级或职业表现。

### 4.2 用户注册与认证

用户体系由思政课平台用户管理统一建设和签发身份，本项目不另建注册、短信和实名核验系统。

- **注册用户：** 由既有思政课平台用户管理系统完成手机号验证及姓名、工作单位登记，鲁韵项目只消费平台令牌与身份声明。
- **认证用户：** 由既有思政课平台通过教育/中心统一身份、组织通讯录、合规核身或邀请机制升级，鲁韵项目据此授予教学业务权限。

鲁韵应用只校验平台下发的用户、单位、角色和认证等级等身份声明。

## 五、总体业务架构

平台采用“任务入口 + 教学成果 + 可信知识 + 专家规则 + 本地模型 + 质量治理”的业务架构。

教师从任务入口发起查依据、备课、诊断或导出。系统根据用户确认的学段、教材、班情和成果上下文，执行资料检索、结构规划、分块生成、规则校验和人工确认，并保存全过程版本与引用。

### 5.1 受控任务能力

平台把检索、对齐、蓝图、分块生成、诊断、修订和导出等能力封装为受控任务单元。每个任务单元具有固定输入输出、版本、权限、配额、超时、降级和审计规则，避免开放无边界工具调用。

### 5.2 教学工作记忆

经用户明确保存和确认后，平台可复用默认学段、教材版本、导出模板、命名班情和常用成果。用户可查看、修改、导出和清除这些内容。平台不把聊天全文静默拼接为长期画像，不将工作记忆用于教师评分、排名或未经授权的模型训练。

## 六、核心应用能力

### 6.1 智能问答与可信资料查询

- 支持政策、教材、课程标准、教学设计和教研类即时问答。
- 优先检索审核有效的资料并展示文件、章节、页码或段落来源。
- 对过期、冲突或资料不足的情况给出明确提示，不编造政策原文。
- 保留历史会话和轻量兼容入口，但复杂成果进入教学项目工作区。

### 6.2 教学设计生成

- 引导教师提供学段、教材、课题、课时、班情、教学重点和实施条件。
- 先生成课标对齐卡和“目标—证据—任务—评价”蓝图，再分块生成正文。
- 支持章节锁定、局部重生成、撤销、版本对比和修改前后差异。
- 支持生成任务单、学习单、问题链、评价量规、板书和课件提纲。
- 按客户模板导出 Word、PDF 或可复制文本。

### 6.3 教学设计诊断

- 支持粘贴或上传已有教案，并允许教师校正结构识别结果。
- 从课程依据、教学目标、学情适配、活动设计、教—学—评一致性、价值与学理统一、评价证据和实施可行性等维度诊断。
- 每条诊断提供原文位置、适用规则、可能影响、修改动作和局部示例。
- 教师可逐条采纳、忽略、编辑或申请专家复核，只对采纳项生成修改稿。
- 诊断对象是教学设计文本及其证据，不输出教师总分、排名或绩效结论。

### 6.4 大中小学一体化专题

- 生成小学、初中、高中、大学的目标递进、核心问题、任务和评价证据。
- 检查相邻学段之间的重复、越级、断裂和“仅并列四份教案”的假一体化。
- 支持山东地方案例在不同学段的任务转换。
- 形成跨学段联合教研方案和一体化专题成果。

### 6.5 课堂活动、评价与教研

- 生成导入语、问题链、案例辨析、角色任务、小组活动和实践任务。
- 生成自评、互评、教师观察和形成性评价工具。
- 支持集体备课、同课异构、听评课记录分析、教研纪要和后续任务。
- 评价工具服务课堂改进，不形成教师或学生思想状态评分。

### 6.6 资源库、知识库与研修

- 资源库面向教师下载、收藏、复用和审核发布；知识库面向模型检索和引用。
- 支持资料上传、解析、去重、版本、有效期、权限、审核、停用和重新索引。
- 支持政策学习、案例研修、学习摘要、讨论问题、测验题和培训材料生成。
- 公开资源必须经过本人授权和专家审核。

### 6.7 管理端与质量运营

- 管理组织、角色、用户、资源、知识、模板、审核和反馈。
- 展示激活、任务完成、保存、导出、复用、资料不足和质量问题。
- 展示队列、容量、失败、降级和恢复状态。
- 运营数据用于改进服务和安排教研，不展示教师个人排名。

### 6.8 微信小程序端（合同硬交付）

小程序在阶段 3 建设，是网页端的轻量延伸，必须完成：

- 思政课平台统一登录及角色、权限、配额继承。
- 可信轻问答和简短教学片段生成。
- 历史会话、收藏、通知和异步任务进度。
- 从小程序发起任务并跳转网页继续复杂编辑。
- 生成内容标识、操作审计和跨端任务一致性。

小程序不承载大文档复杂编辑、完整工作记忆管理、资源审核、管理看板和用户自建开放工具。

### 6.9 受控多智能体、多模态与模型优化

阶段 4 在门禁通过的前提下建设：

- 教学设计协同生成、教案诊断与证据核验、多模态实施反思三条受控流程。
- 图片/扫描件区域定位、课堂录音时间戳和授权短视频关键证据定位。
- 教师可查看原始区域或时间点并纠正低置信结果。
- 多智能体具备工具白名单、最大轮次、超时、预算、人工确认和单工作流回退。
- 微调达到数据、质量、安全和硬件门槛时交付模型；不达标时交付完整实验和正式不采纳报告。

平台不建设面部识别、情绪识别、课堂常态监控、思想状态推断和教师绩效评分。

## 七、教学与课程论专业体系

平台使用四层教学成果结构：

| 层级 | 核心问题 | 主要成果 |
| --- | --- | --- |
| 课程层 | 为什么教、培养什么、学段如何安排 | 课程实施档案、内容地图、素养发展和学段衔接 |
| 单元层 | 多个课时如何围绕核心问题形成完整学习过程 | 单元目标、核心议题、表现性任务、证据和课时序列 |
| 课时层 | 学生经历什么、产生什么学习证据 | 目标、情境、任务链、支架、评价、作业和板书 |
| 实施反思层 | 实际发生了什么、下一轮如何调整 | 实施证据、偏差原因、局部修改和下一版本 |

系统不得替教师虚构班情。缺失的学生基础、设备、时间和实施条件必须标记为“假设/待确认”。

## 八、可信知识与模型能力

### 8.1 知识来源

资料按正式政策和课程标准、授权教材教参、中心审核成果、地方权威资源、教师个人资料和 AI 建议分级管理。权威回答只使用审核有效且权限允许的资料，AI 生成内容不能反向成为权威来源。

### 8.2 模型使用原则

本地模型主要承担语义理解、规划、解释和局部生成，不单独决定课程标准有效性、学段边界、引用合法性、公开发布和高风险结论。

优化顺序为：结构化约束与工作流、可信检索与重排、提示词和专家规则、LoRA/SFT。领域继续预训练和偏好优化只在数据、权利、质量和业务收益充分时评估，不以“做过训练”作为验收目标。

正式环境和最终验收默认使用经验证的固定版本 vLLM 推理服务。业务通过模型网关使用逻辑模型名，不绑定具体模型目录或推理引擎。

## 九、评测、质量与安全治理

### 9.1 评测体系

平台建立开发集、CI 回归集、密封验收集和线上事件集四套隔离评测库。密封验收集由独立 QA/专家保管，不用于训练、调参或提示词开发。

评测覆盖：资料解析与检索、引用支持、教学设计、诊断、思政专业质量、任务体验、运行性能、安全和恢复。关键风险采用双专家标注、仲裁和一致性报告；自动评审不能独立决定政治、教材、引用、安全和隐私门槛。

### 9.2 关键质量原则

- 伪造引用、重大政治/政策/教材事实错误、越权访问和重大隐私事件为一票否决项。
- 每个新增学段、课型和教学模式单独验收，不以总体平均值掩盖局部失败。
- 资料不足时必须正确拒答或降级为一般教学建议。
- 多智能体必须相对单工作流带来可验证收益，否则回退单工作流。

### 9.3 安全与合规

- 建立生成式人工智能服务、数据、个人信息、教育场景和内容标识适用性矩阵。
- 页面、复制和导出内容按适用要求提供 AI 生成/辅助显式标识和文件元数据隐式标识。
- 提供引用纠错、投诉举报、个人信息查阅、更正和删除流程。
- 每个阶段门执行独立红队，覆盖提示注入、信息泄漏、供应链、投毒、不安全输出、Agent 越权、向量检索、错误信息和资源耗尽。
- 形成 AI 事故分级、停止生成、下架、报告、复盘和恢复机制。

## 十、部署与运行保障

正式生产环境部署在客户校内或客户确认的合规环境，模型、资料、成果和日志不静默发送到外部模型服务。

平台采用网页入口、应用服务、异步任务、数据库、对象存储、检索服务、模型网关和本地推理服务分层架构。短问答和检索实时返回，长教案、诊断、解析和导出进入异步队列。

每次正式发布绑定应用、镜像、模型、提示词、规则、知识、数据、任务能力和导出模板版本；执行专业回归、安全检查、数据库迁移、备份恢复和回滚验证。系统建立服务 SLO、知识更新 SLO、RTO/RPO、容量和单位任务成本监控。

## 十一、实施计划

建议合同完整周期为 46—63 周。专家、资料、平台接口和硬件均按期到位时争取 46—55 周；若关键依赖延迟，不以压缩评测、试点、安全和恢复演练换取工期。

| 阶段 | 周期 | 主要建设内容 | 决策门 |
| --- | ---: | --- | --- |
| 阶段 0 联合定义与基线 | 5—7 周 | 专业模型、量规、资料授权、合规、评测四库、红队、模型硬件和原型 | G0 是否具备全面开发条件 |
| 阶段 1 可信底座与纵向样板 | 9—12 周 | 1A 平台骨架；1B 查依据、备课、诊断、导出纵向样板 | G1 样板是否值得扩展 |
| 阶段 2 生产与诊断闭环 | 8—10 周 | 完整生成、诊断、逐条修改、导出、专家回归和真实试点 | G2 是否达到受控开放条件 |
| 阶段 3 一体化与生产运营 | 10—14 周 | 四学段、一体化、资源、权限、看板、小程序和生产加固 | G3 是否扩大区域开放 |
| 阶段 4 智能协作与多模态增强 | 14—20 周 | 受控多智能体、多模态证据、微调路径 A/B 和综合试点 | G4 合同最终验收 |

每个阶段以质量决策门为进入下一阶段的条件，不以日期自动放行。

## 十二、主要交付成果

- 教学与课程论领域模型、教学设计结构和诊断量规。
- 可信知识库、引用服务、资料授权和知识运营制度。
- 教学成果工作台、受控任务能力和教学工作记忆。
- 教学设计生成、诊断、局部修改、版本对比和标准导出。
- 四学段能力及大中小学一体化专题。
- 资源审核、管理看板、组织权限和小程序轻量端。
- 专家评测集、独立红队、Data Card、System Card 和发布门禁。
- 受控多智能体、多模态证据及微调 Go/No-Go 决策包。
- 私有化部署、监控、备份、恢复、回滚、培训和运维材料。

## 十三、验收框架

| 验收领域 | 核心要求 |
| --- | --- |
| 可信问答 | 正确展示来源；资料不足不编造；权限过滤无跨组织泄漏 |
| 教学设计 | 结构完整、依据可追溯、学段适配、可编辑、可导出 |
| 教案诊断 | 问题有原文证据和规则依据；建议可执行；不对教师评分排名 |
| 一体化 | 体现目标、内容、任务和证据的真实递进，不是四份教案拼接 |
| 小程序 | 统一登录、轻问答、收藏/历史、通知、任务进度、跨端续办、标识和审计 |
| 安全合规 | 权限、隐私、标识、投诉纠错、独立红队和事故流程通过 |
| 模型与增强能力 | 固定版本、可追溯、可降级；Agent/多模态/微调分别通过门禁或形成正式 No-Go |
| 运行保障 | 长任务、容量、监控、备份、RTO/RPO 和回滚达到签字指标 |

最终数值门槛、样本量和统计口径在阶段 0 由建设单位、专家组和承建单位共同冻结，并作为 G1—G4 验收附件。

## 十四、客户配合与责任边界

客户需提供或确认：四学段专家和仲裁负责人；课程标准、教材教参、中心资料和地方资源授权；标准 Word 模板和代表性教案；平台身份、组织、资源和审计接口；试点学校和教师；GPU、存储、网络和备份条件；资源审核、投诉和事故责任人；小程序发布主体及平台能力；多模态采集、保留、删除和训练授权边界。

用户注册、短信、认证升级和用户中心由思政课平台用户管理建设。鲁韵项目建设教学成果、任务能力、工作记忆、知识检索、模型调用、诊断、资源和质量治理。

## 十五、持续服务

平台上线后，围绕“资料、规则、评测、反馈、运行”持续运营：

- 按政策、课程文件和教材变化更新资料并验证引用。
- 按专家仲裁和试点结果更新量规、模板和任务能力。
- 对模型、规则、知识和应用版本持续回归和独立抽检。
- 对资料不足、错误、投诉、安全事件和容量问题闭环处理。
- 定期开展教师、专家、内容运营和技术运维培训。
