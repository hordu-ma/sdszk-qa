# 鲁韵思政大模型 2026 功能拓展开发建议

> **文档定位：功能方向基础稿，非排期依据。**  
> 合同范围、阶段工作包、验收、产品 Skills、核心用户 Memory、用户注册/认证、工程顺序以  
> `2026-luyun-curriculum-pedagogy-development-plan.md`（v1.0）为**唯一主依据**。
> 当前仓库已包含问答 MVP 和阶段 1A 首个可部署增量；本文是方向建议而非排期，功能状态以主开发计划为准。
> 已确认合同覆盖阶段 0–4；阶段 4 多智能体、多模态、微调为**门禁式**交付（路径 A/B）。  
> **用户注册/认证在思政课平台用户管理开发，不在本仓开发。**

## 结论

鲁韵思政大模型 2026 年升级不应继续停留在单一问答机器人形态，而应升级为面向山东省大中小学思政课教师、教研员和管理者的“思政教学支持工作台”。

平台应：

- 保留轻量问答兼容能力。
- 以**教学成果项目**为主对象。
- 以**产品 Skills** 为任务执行入口（受控、可审计）。
- 以**核心用户 Memory** 为显式工作上下文（可删、非侧写）。
- 以知识库增强、结构化生成、证据化诊断、导出与试点闭环为阶段 1–2 主线。
- 再扩展四学段一体化、运营与阶段 4 增强能力。

部署原则：高频轻量实时、复杂任务异步、资源缓存、管理可观测、**网页桌面优先**、阶段 3 小程序轻量端硬交付；避免把所有能力压到实时大模型链路。

## 一、升级背景

当前 MVP 已具备登录认证、主题列表、会话创建、SSE 流式问答、历史会话和审计日志等基础能力。

主要短板：

- 通用问答为主，缺少备课/诊断工作流与成果对象。
- 缺少可追溯知识库与引用。
- 缺少产品 Skills 与显式 Memory，上下文依赖隐式会话。
- 缺少异步队列、额度、降级与发布回归门禁。
- 前端偏轻交互，不足以支撑长文档工作台。
- 中心需要教研与治理能力，而不仅是个人聊天工具。

## 二、产品定位建议

### 2.1 平台定位

> 面向山东省大中小学思政课教师、教研员和管理者的省级思政教学智能支持平台。

开发落点：

> 先帮助教师完成查依据、形成可改设计、诊断已有教案、导出课堂成果；Skills 执行任务，Memory 减少重复输入，专家规则与 RAG 约束专业边界。

### 2.2 用户角色

- 一线教师（P0）
- 中心专家/审核员（P0）
- 教研员（P1）
- 中心管理员（P1）
- 技术管理员（P0 支撑）
- 学生：合同全周期不开放生成式服务

### 2.3 端侧分工

- **网页桌面工作台：** Skills 任务、Memory 管理、生成/诊断/版本/导出、资源与管理。
- **小程序（合同硬交付，阶段 3）：** 统一登录、可信轻问答、收藏/历史、通知、进度摘要；跳转网页继续复杂编辑。

## 三、功能模块开发建议

### 3.0 用户注册与认证（P0，平台用户管理）

- **注册用户**：手机验证 + 姓名 + 工作单位（步骤 1–4）；可登录使用（配额可收紧）。
- **认证用户**：步骤 5（SSO / 三要素 / 邀请或通讯录等）后升级。
- 不称自填为实名；默认不采未核验身份证号。
- **开发在思政课平台用户管理**；鲁韵只对接 claims。详见主计划 §2.6。

### 3.1 产品 Skills（P0，阶段 0 规格 / 阶段 1 运行时）

- 注册表 + SkillRun 审计 + Schema。
- 阶段 1 最小集：`retrieve_basis`、`alignment_card`、`design_blueprint`、`generate_section`、`diagnose_design`、`apply_revision`、`export_artifact`、`manage_memory`。
- 阶段 4 Agent 只调 Skills 白名单。

### 3.2 核心用户 Memory（P0，阶段 0 边界 / 阶段 1 最小集）

- 偏好、命名班情、钉选模板/成果、注入确认、清除、注入审计。
- 禁止：思想侧写、能力评级、教师排名、静默拼历史全文、未授权训练。

### 3.3 智能问答与可信检索

- 兼容现有 SSE 问答。
- 知识库检索优先；区分“基于资料”与“模型建议”。
- 资料不足明确提示，禁止伪造引用。

### 3.4 备课与诊断闭环

- 先结构蓝图后正文；非评分诊断；逐条采纳与局部修订；标准 Word 导出。
- 复杂任务异步。

### 3.5 教研 / 资源 / 研修 / 管理

- 按主计划阶段 2–3 扩展；不得挤占阶段 1 纵向样板。
- 资源库与知识库分离。
- 管理看板不展示教师个人排名。

## 四、负载控制与部署建议

| 类型 | 建议交互 | 负载策略 |
| --- | --- | --- |
| 高频短问答 / 检索 Skill | 实时 SSE | 短上下文，限流 |
| 教案生成 / 诊断 / 导出 | 异步任务 | 排队、可取消 |
| Memory 读写 | 实时轻量 API | 严格权限 |
| 管理统计 | 离线汇总 | 低峰 |
| 小程序 | 更严限流 | 阶段 3 硬交付，轻量边界 |

- 正式/验收默认 vLLM；Ollama 仅开发/过渡/明示降级。
- ModelGateway + 逻辑模型名；OpenClaw 不进鲁韵链路。
- `luyun-int` → 门禁 → 同镜像 `luyun-demo`；演示不抢跑。

## 五、分期建设路线

与主计划一致（摘要）：

| 阶段 | 目标 |
| --- | --- |
| 0 | 专业框架 + Skills/Memory + 合规 + 评测四库/红队 + 硬件/演示基线 |
| 1 | 1A 可信平台骨架 + 1B Skills/Memory 纵向样板 |
| 2 | 完整闭环与试点 |
| 3 | 四学段、一体化、运营；小程序轻量端硬交付 |
| 4 | 门禁化多智能体/多模态/微调评估 |

工程顺序见主计划 §5.4.1。合同建议总周期 46–63 周。

## 六、优先级建议

**P0（阶段 0–2 主线）：**

- 平台用户管理：注册用户 / 认证用户（**非本仓**）。
- Skills 目录与运行时。
- Memory 边界与最小集。
- 知识库问答带引用。
- 教学成果对象与结构化生成。
- 证据化诊断（非评分）与导出。
- 异步任务、限流、审计。
- 服务层抽离与 ModelClient。
- 本仓对接平台身份 claims（对接期，非自建注册）。

**P1：**

- 四学段与一体化。
- 资源库与审核。
- 管理看板。
- 小程序轻量端（阶段 3 合同硬交付）。

**P2 / 后置增强：**

- 集体备课深度协同、研修档案等。

**阶段 4 硬交付但门禁化（非“无条件上线”）：**

- 受控多智能体（复用 Skills）。
- 多模态证据定位与纠错。
- 微调 A/B 决策包。

## 七、风险与注意事项

- 不按十二模块并行开工。
- 不把演示双环境建设抢在样板专业能力之前过度投资。
- 不把隐式聊天历史当作 Memory。
- 不做教师/学生评分排名产品。
- 不开放无白名单 Agent 工具。
- 不把小程序当完整工作台，但必须按阶段 3 合同边界交付。
- 无引用不回答确定性政策口径。
