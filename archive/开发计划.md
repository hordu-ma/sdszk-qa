## 📋 **项目概述**

### 核心目标

构建一个用于**临床医学教学**的模拟问诊系统：

- 学生负责诊断
- 大模型**始终扮演病人**
- 本地 Win11（RTX Pro 500）开发
- 生产环境 A/B 双机部署
- 全流程 Docker 化，数据可审计

### 架构概览

- **A 服务器（入口层）**：Nginx（HTTPS/SSE 反代）+ 静态前端
- **B 服务器（核心层，GPU）**：FastAPI + vLLM + PostgreSQL + MinIO

### 核心原则

- 后端**只依赖 OpenAI 兼容接口**
- 推理服务地址通过 `LLM_BASE_URL` 切换
- 开发/生产架构保持一致
- 所有用户行为与结果**可审计、可复现**
- **用户数据与其他系统共用，无注册功能**

3. **创建依赖注入模块**（`src/apps/api/dependencies.py`）

   ```python
   from typing import Annotated
   from collections.abc import AsyncGenerator

   from fastapi import Depends, HTTPException, status
   from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
   from sqlalchemy.ext.asyncio import AsyncSession, async_sessionmaker, create_async_engine
   import jwt
   from .config import settings
   from .models import User

   security = HTTPBearer()

   # 异步数据库引擎（将 psycopg 驱动替换为 asyncpg）
   async_db_url = settings.DATABASE_URL.replace("postgresql+psycopg://", "postgresql+asyncpg://")
   engine = create_async_engine(async_db_url, echo=settings.DEBUG, pool_pre_ping=True)
   AsyncSessionLocal = async_sessionmaker(engine, class_=AsyncSession, expire_on_commit=False)

   async def get_db() -> AsyncGenerator[AsyncSession, None]:
       async with AsyncSessionLocal() as session:
           try:
               yield session
           finally:
               await session.close()

   async def get_current_user(
       credentials: Annotated[HTTPAuthorizationCredentials, Depends(security)],
       db: Annotated[AsyncSession, Depends(get_db)],
   ) -> User:
       token = credentials.credentials
       try:
           payload = jwt.decode(token, settings.JWT_SECRET, algorithms=[settings.JWT_ALGORITHM])
           user_id = int(payload.get("sub"))
       except (jwt.ExpiredSignatureError, jwt.InvalidTokenError, ValueError):
           raise HTTPException(status_code=401, detail="Invalid or expired token")

       # 从数据库加载完整用户对象
       from sqlalchemy import select
       result = await db.execute(select(User).where(User.id == user_id))
       user = result.scalar_one_or_none()
       if user is None:
           raise HTTPException(status_code=404, detail="User not found")
       if not user.is_active:
           raise HTTPException(status_code=403, detail="User is inactive")
       return user

   # 类型别名，简化路由函数签名
   DbSession = Annotated[AsyncSession, Depends(get_db)]
   CurrentUser = Annotated[User, Depends(get_current_user)]
   ```

```python
async def chat_stream(session_id: str, message: str, user_id: int):
    # 1. 权限检查：验证用户是否拥有该会话
    session = await get_session(session_id)
    if session.user_id != user_id:
        raise HTTPException(403)

    # 2. 加载病例数据
    case = await get_case(session.case_id)

    # 3. 构造 Prompt
    system_prompt = "你是一名患者，正在接受医生问诊。你不能做诊断，只能根据自身症状回答医生的问题。"
    developer_prompt = f"""
    病例信息：
    - 主诉：{case.chief_complaint}
    - 现病史：{case.present_illness}
    - 可透露体征：{case.physical_exam['on_request']}

    注意：
    1. 只回答医生直接询问的内容
    2. 不要主动提供未询问的信息
    3. 不要使用医学术语，用患者语言描述
    """

    # 4. 加载历史消息（最近 10 轮或摘要）
    history = await get_recent_messages(session_id, limit=20)

    # 5. 组装请求
    messages = [
        {"role": "system", "content": system_prompt},
        {"role": "developer", "content": developer_prompt},
        *history,
        {"role": "user", "content": message}
    ]

    # 6. 调用 vLLM（流式）
    async with httpx.AsyncClient() as client:
        response = await client.post(
            f"{settings.LLM_BASE_URL}/v1/chat/completions",
            json={
                "model": settings.LLM_MODEL,
                "messages": messages,
                "stream": True,
                "temperature": 0.7,
                "max_tokens": 500
            },
            timeout=60.0
        )

        # 7. SSE 流式返回
        async def event_generator():
            full_response = ""
            start_time = time.time()

            async for line in response.aiter_lines():
                if line.startswith("data: "):
                    data = line[6:]
                    if data == "[DONE]":
                        break
                    chunk = json.loads(data)
                    content = chunk["choices"][0]["delta"].get("content", "")
                    full_response += content
                    yield f"data: {json.dumps({'content': content})}\n\n"

            # 8. 流式结束，落库
            latency = int((time.time() - start_time) * 1000)
            await save_messages(session_id, [
                {"role": "user", "content": message, "tokens": count_tokens(message)},
                {"role": "assistant", "content": full_response, "tokens": count_tokens(full_response), "latency": latency}
            ])

            yield "data: [DONE]\n\n"

        return StreamingResponse(event_generator(), media_type="text/event-stream")
```

1. **评分逻辑设计**（`src/apps/api/services/scoring.py`）

   **评分维度**：

   - **红旗症状识别**（20%）：是否询问危险症状
   - **检查合理性**（20%）：申请检查是否恰当
   - **诊断准确性**（20%）：学生提交的诊断与标准答案匹配度

   **实现方法**：

   ```python
   def calculate_score(session: Session, case: Case, submitted_diagnosis: str) -> dict:
       # 1. 从消息历史提取关键词
       messages = get_session_messages(session.id)
       keywords_asked = extract_keywords(messages)

       # 2. 与病例 key_points 对比
       key_points_coverage = len(set(keywords_asked) & set(case.key_points)) / len(case.key_points)

       # 3. 检查申请合理性
       tests = get_session_tests(session.id)
       appropriate_tests = [t for t in tests if t.type in case.recommended_tests]
       test_score = len(appropriate_tests) / max(len(case.recommended_tests), 1)

       # 4. 诊断匹配度（简单关键词匹配）
       diagnosis_match = any(kw in submitted_diagnosis for kw in case.standard_diagnosis["primary"])

       # 5. 综合计算
       total_score = (
           key_points_coverage * 40 +
           test_score * 20 +
           (100 if diagnosis_match else 0) * 0.2
       )

       return {
           "total_score": round(total_score, 2),
           "dimensions": {
               "interview_completeness": round(key_points_coverage * 100, 2),
               "test_appropriateness": round(test_score * 100, 2),
               "diagnosis_accuracy": 100 if diagnosis_match else 0
           },
           "details": {
               "keywords_asked": keywords_asked,
               "key_points_covered": list(set(keywords_asked) & set(case.key_points))
           }
       }
   ```

````
## ✅ 随机病例模式（vLLM + FastAPI）开发结论（2026-01-20）

### 功能实现与行为约定

- **随机病例模式链路已打通**：前端以 `mode=random` 创建会话；后端生成病例 JSON 后落库（`cases.source=random`），并创建会话用于后续问诊/评分。
- **随机病例不出现在病例列表**：`GET /api/cases` 默认仅展示固定病例（避免暴露标准答案与污染训练集）。

### 本地开发环境联通性

- **vLLM 启动成功**：通过 `src/scripts/start_vllm_dev.sh` 启动本地 vLLM（8001），`/v1/models` 可返回模型信息。
- **Docker API 可访问宿主机 vLLM**：dev compose 已配置 `extra_hosts: host.docker.internal:host-gateway`，并在 `.env` 中使用 `LLM_BASE_URL=http://host.docker.internal:8001`。
- **修复容器内 DB 连接**：`.env` 的 `DATABASE_URL` 应指向 compose 网络内服务名（`postgres`），避免容器内访问 `localhost:5432` 失败。

### 关键接口返回结构（E2E 脚本注意事项）

- `GET /api/cases/{case_id}/available-tests` 返回结构为对象而非数组：

```json
{
 "case_id": 13,
 "items": [{ "type": "x_ray", "name": "..." }],
 "total": 1
}
````

E2E 取值应使用 `resp["items"][0]["type"]`。

### 稳定性修复（随机病例生成数据归一化）

- 修复 `POST /api/sessions/{session_id}/tests` 500：随机病例生成的 `available_tests[].result` 可能为字符串，后端 schema 要求为 dict；已在病例生成服务中做归一化（缺失→`{}`，非 dict→`{"summary": "..."}`）。

### E2E 验证结果

- 已完成真实链路 E2E：登录 → 创建随机会话 → 对话（SSE）→ 获取可用检查 → 申请检查 → 提交诊断 → 获取评分 → 会话列表确认。

### 已知问题（非阻塞）

- `mode=random` 创建会话仍可能偶发 502（LLM 输出非严格 JSON 导致解析失败）；可通过降低温度、增大 `--max-model-len` 并同步 `LLM_MAX_CONTEXT_LEN`、或增加“JSON 修复二次请求”策略进一步提升稳定性。
